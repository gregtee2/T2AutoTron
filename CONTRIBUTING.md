# Contributing to T2AutoTron

Thank you for your interest in contributing! T2AutoTron is an open-source, community-driven project. We welcome all contributions—code, documentation, bug reports, feature requests, and more.

## How to Contribute

### 1. Fork the Repository
- Click "Fork" at the top right of the GitHub page.
- Clone your fork locally:
  ```bash
  git clone https://github.com/<your-username>/T2AutoTron.git
  ```

### 2. Create a Branch
- Use a descriptive branch name:
  ```bash
  git checkout -b feature/my-new-feature
  ```

### 3. Make Your Changes
- Follow the coding style and patterns in the project.
- Add tests for new features or bug fixes if possible.
- Update documentation as needed.

### 4. Commit and Push
- Write clear, concise commit messages.
- Push your branch to your fork:
  ```bash
  git push origin feature/my-new-feature
  ```

### 5. Open a Pull Request
- Go to the original repo and click "New Pull Request."
- Fill out the PR template and describe your changes.

## Code of Conduct

Please be respectful and constructive. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for details.

## Reporting Bugs & Requesting Features
- Use [GitHub Issues](https://github.com/gregtee2/T2AutoTron/issues) for bug reports and feature requests.
- Include clear steps to reproduce, screenshots, and environment details.

## Style Guide
- Use consistent formatting (see `.eslintrc` and Prettier config).
- Write descriptive comments for complex logic.
- Prefer small, focused commits.

## Need Help?
- Start a [Discussion](https://github.com/gregtee2/T2AutoTron/discussions) for questions or ideas.
- Join the community—feedback and suggestions are always welcome!
