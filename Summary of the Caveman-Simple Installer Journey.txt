Summary of the Caveman-Simple Installer Journey
What We Set Out to Do
Your goal was to create a .deb installer for T2AutoTron that:
Assumes a Raspberry Pi 5 with only the pre-flashed Raspberry Pi OS (Bookworm, 64-bit).

Automatically installs all dependencies (Node.js, MongoDB, Home Assistant, etc.).

Requires minimal user input (just running the .deb and adding tokens).

Is secure, user-friendly, and robust, like a "caveman could do it" (inspired by GEICO’s simplicity).

The app integrates Home Assistant for smart home control (e.g., Shelly, Hue, Z-Wave/Matter via API), MongoDB for data, Telegram for notifications, and an Electron/LiteGraph UI, all tailored for the Pi 5’s ARM64 architecture.
What We Did
Initial Setup and Dependency Management:
Started with your package.json, which included dependencies like electron@33.2.1, express, mongoose, node-telegram-bot-api, shelly-driver, and Home Assistant integrations (node-hue-api, shellies-ng, mqtt).

Identified vulnerabilities (e.g., axios@0.21.4, ip, request, tough-cookie) via npm audit (initially 12 vulnerabilities).

Fixed vulnerabilities by:
Upgrading to axios@1.9.0 and enforcing it via overrides and resolutions.

Replacing bonjour with bonjour-service@1.2.1 to resolve ip issues (via dns-packet@5.6.1).

Removing request-related dependencies (@cypress/request-promise, request-promise, request-promise-core) to eliminate request/tough-cookie vulnerabilities.

Upgrading node-telegram-bot-api to 0.66.0 (tested and confirmed working).

Achieved 0 vulnerabilities (npm audit clean).

Installer Configuration:
Used electron-builder to create a .deb package targeting Linux ARM64.

Added before-install.sh to:
Verify the OS (Raspberry Pi OS, Bookworm).

Install Node.js 20.x and MongoDB.

Added after-install.sh to:
Configure GPIO permissions.

Create a desktop shortcut.

Install Home Assistant Core (2025.5.5).

Set up systemd services for the backend and Home Assistant.

Open port 8123 for Home Assistant.

Prompt for Home Assistant and Telegram tokens.

Created .env with placeholders for MONGO_URI, HOME_ASSISTANT_TOKEN, and TELEGRAM_BOT_TOKEN.

Challenges Overcome:
Vulnerabilities: Iteratively resolved 12 vulnerabilities through dependency updates, overrides, and removals.

Windows Build Issues:
Fixed electron in dependencies (moved to devDependencies).

Removed invalid architecture property from build.linux.

Added missing homepage field.

Encountered fpm missing error, prompting the switch to build on the Pi 5.

Line Endings: Ensured before-install.sh and after-install.sh use Unix LF endings for Linux compatibility.

Telegram: Confirmed node-telegram-bot-api@0.66.0 compatibility, avoiding breaking changes.

What You Need to Do Next
The next steps involve building the .deb on the Pi 5, testing the installer, and distributing it. Below are the detailed instructions, followed by optional enhancements.
Next Steps on the Pi 5
1. Transfer the Project to the Pi 5
Action:
Copy the project directory to the Pi:
powershell

scp -r C:\X_T2_AutoTron2.0 pi@<pi-ip-address>:/home/pi/t2-autotron

Replace <pi-ip-address> with your Pi’s IP (find it with hostname -I on the Pi or check your router).

Verify Transfer:
On the Pi:
bash

ls /home/pi/t2-autotron

Confirm files like package.json, scripts/before-install.sh, scripts/after-install.sh, .env, and src/ are present.

2. Set Up the Build Environment on the Pi
Action:
Install dependencies for building:
bash

sudo apt update
sudo apt install -y ruby ruby-dev rubygems build-essential
sudo gem install fpm

Verify fpm:
bash

fpm --version  # Should show version (e.g., 1.15.0)

Install Node.js 20.x (if not already installed):
bash

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v  # Should show v20.x.x
npm -v

Install Git (optional, for version control):
bash

sudo apt install -y git

3. Build the Installer
Action:
Navigate to the project:
bash

cd /home/pi/t2-autotron

Install npm dependencies:
bash

npm install

Run the build:
bash

npm run build

This should create /home/pi/t2-autotron/dist/t2-autotron_2.0.0_arm64.deb.

Verify Output:
bash

ls dist

Confirm t2-autotron_2.0.0_arm64.deb exists.

If Build Fails:
Check the error message (e.g., missing dependencies, file permissions).

Share the error log (e.g., from /home/pi/t2-autotron/node_modules/.cache/electron-builder/logs).

Common fixes:
Ensure sufficient disk space:
bash

df -h

Clear npm cache:
bash

npm cache clean --force
npm install

4. Test the Installer
Action:
Install the .deb:
bash

sudo dpkg -i /home/pi/t2-autotron/dist/t2-autotron_2.0.0_arm64.deb
sudo apt install -f

This runs before-install.sh (installs Node.js, MongoDB), installs the app, and runs after-install.sh (sets up Home Assistant, systemd services).

Configure tokens:
bash

sudo nano /usr/lib/t2-autotron/.env

Update:
bash

HOME_ASSISTANT_TOKEN=your_token_here
TELEGRAM_BOT_TOKEN=your_bot_token_here

Home Assistant Token: Access http://<pi-ip>:8123/profile after setup to generate a long-lived access token.

Telegram Token: Use your bot token from BotFather.

Reboot:
bash

sudo reboot

Verify Components:
Frontend: Launch via the desktop shortcut (created by after-install.sh) or:
bash

/usr/bin/t2-autotron

Confirm the Electron/LiteGraph UI loads and functions (e.g., displays smart home controls).

Backend: Check:
bash

sudo systemctl status t2-autotron-backend

Ensure active (running). View logs if issues:
bash

sudo journalctl -u t2-autotron-backend

Home Assistant: Access:
bash

http://<pi-ip>:8123

Verify setup completes and devices (e.g., Shelly, Hue) are integrated. Check:
bash

sudo systemctl status homeassistant

Telegram: Send a message or command to the bot. Confirm responses (e.g., notifications).

MongoDB: Verify data persistence (e.g., app settings, logs). Check:
bash

sudo systemctl status mongodb

Test on Fresh Pi:
Use a clean Pi 5 with pre-flashed OS to ensure the installer works without prior setup.

Follow the installation steps and report any issues.
