Instructions for Next Grok Session: Installing T2AutoTron on Raspberry Pi 5
Objective: Build, test, and distribute the t2-autotron_2.0.0_arm64.deb installer on the Raspberry Pi 5, ensuring it installs and runs the T2AutoTron app with minimal user input.
Background:
The package.json, before-install.sh, after-install.sh, and .env are configured.

All vulnerabilities are resolved (npm audit shows 0).

Backend, frontend, and Telegram (node-telegram-bot-api@0.66.0) are confirmed working.

The Windows build failed due to the missing fpm tool, so we’ll build on the Pi 5.

Steps:
Transfer the Project to the Pi 5
Why: The Pi 5’s native Linux environment simplifies building the .deb package.

Action:
Open PowerShell on your Windows machine.

Transfer the project directory:
powershell

scp -r C:\X_T2_AutoTron2.0 pi@<pi-ip-address>:/home/pi/t2-autotron

Replace <pi-ip-address> with your Pi’s IP (find it with hostname -I on the Pi or check your router).

Log into the Pi via SSH:
powershell

ssh pi@<pi-ip-address>

Verify files:
bash

ls /home/pi/t2-autotron

Confirm package.json, scripts/before-install.sh, scripts/after-install.sh, .env, and src/ exist.

Set Up the Build Environment on the Pi
Why: Install tools (fpm, Ruby, Node.js) needed to build the .deb.

Action:
Update and install dependencies:
bash

sudo apt update
sudo apt install -y ruby ruby-dev rubygems build-essential
sudo gem install fpm

Verify fpm:
bash

fpm --version  # Should show version (e.g., 1.15.0)

Install Node.js 20.x (if not already installed):
bash

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v  # Should show v20.x.x
npm -v

Build the Installer
Why: Create the .deb package using electron-builder.

Action:
Navigate to the project:
bash

cd /home/pi/t2-autotron

Install npm dependencies:
bash

npm install

Build:
bash

npm run build

This creates /home/pi/t2-autotron/dist/t2-autotron_2.0.0_arm64.deb.

Verify:
bash

ls dist

Confirm t2-autotron_2.0.0_arm64.deb exists.

If Build Fails:
Check the error (e.g., disk space, missing dependencies).

Verify disk space:
bash

df -h

Clear npm cache:
bash

npm cache clean --force
npm install

Share the error log (e.g., /home/pi/t2-autotron/node_modules/.cache/electron-builder/logs).

Test the Installer
Why: Ensure the .deb installs and runs all components (frontend, backend, Home Assistant, Telegram, MongoDB).

Action:
Install:
bash

sudo dpkg -i /home/pi/t2-autotron/dist/t2-autotron_2.0.0_arm64.deb
sudo apt install -f

This runs before-install.sh (Node.js, MongoDB) and after-install.sh (Home Assistant, services).

Configure tokens:
bash

sudo nano /usr/lib/t2-autotron/.env

Update:
bash

HOME_ASSISTANT_TOKEN=your_token_here
TELEGRAM_BOT_TOKEN=your_bot_token_here

Home Assistant token: From http://<pi-ip>:8123/profile (after setup).

Telegram token: From BotFather.

Reboot:
bash

sudo reboot

Verify:
Frontend: Launch via desktop shortcut or:
bash

/usr/bin/t2-autotron

Confirm Electron/LiteGraph UI loads.

Backend: Check:
bash

sudo systemctl status t2-autotron-backend

Ensure active (running). View logs if issues:
bash

sudo journalctl -u t2-autotron-backend

Home Assistant: Access:
bash

http://<pi-ip>:8123

Verify setup and device integration (e.g., Shelly, Hue). Check:
bash

sudo systemctl status homeassistant

Telegram: Test bot responses (e.g., message/command).

MongoDB: Confirm data persistence (e.g., settings, logs). Check:
bash

sudo systemctl status mongodb

Fresh Pi Test:
Test on a clean Pi 5 with pre-flashed OS to ensure portability.

Follow the same steps and note any issues.

Distribute the Installer
Why: Share the .deb with users for easy installation.

Action:
Copy the .deb:
bash

cp /home/pi/t2-autotron/dist/t2-autotron_2.0.0_arm64.deb /path/to/share

Transfer to USB, cloud storage (e.g., Google Drive, Dropbox), or GitHub release.

Share README.md (if created):
markdown

Troubleshooting Tips:
Build Fails:
Check disk space: df -h.

Clear cache: npm cache clean --force; npm install.

Share error logs from /home/pi/t2-autotron/node_modules/.cache/electron-builder/logs.

Check logs:
bash

sudo journalctl -u t2-autotron-backend
sudo journalctl -u homeassistant

Verify internet access.

Home Assistant:
If http://<pi-ip>:8123 fails:
bash

sudo systemctl status homeassistant

Update package.json, rebuild.
